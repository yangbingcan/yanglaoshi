# Code Reviewer

语言无关的通用代码审查技能，提供专业的代码质量检查和改进建议。

## 触发条件

- 用户要求代码审查或 Code Review
- 用户需要检查代码质量
- 用户需要发现潜在 Bug
- 用户需要代码优化建议

## 审查维度

### 1. 代码质量
- 可读性：命名规范、代码结构、注释质量
- 可维护性：模块化、单一职责、耦合度
- 一致性：编码风格、格式规范

### 2. 潜在问题
- 逻辑错误
- 边界条件处理
- 异常处理
- 资源泄漏
- 安全漏洞

### 3. 性能考量
- 算法复杂度
- 内存使用
- I/O 效率
- 缓存策略

### 4. 最佳实践
- 设计模式应用
- SOLID 原则
- DRY 原则
- KISS 原则

## 审查输出格式

```markdown
## 代码审查报告

### 概述
- 文件/模块名称
- 审查范围
- 整体评分

### 发现的问题

#### 严重问题 (Critical)
- [问题描述] - 位置：行号 - 建议：[修复方案]

#### 重要问题 (Major)
- [问题描述] - 位置：行号 - 建议：[修复方案]

#### 建议改进 (Minor)
- [问题描述] - 位置：行号 - 建议：[修复方案]

### 优点
- [代码亮点]

### 总结
- [整体评价和改进建议]
```

## 审查原则

1. **客观公正**：基于事实和最佳实践
2. **建设性**：提供具体可行的改进建议
3. **优先级明确**：区分严重程度
4. **尊重上下文**：考虑项目实际情况
