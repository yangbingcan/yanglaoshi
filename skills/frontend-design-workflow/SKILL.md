# Frontend Design Workflow

前端设计稿生成与管理的全链路解决方案。

## 触发条件

- 用户需要设计规范
- 用户需要设计到代码的工作流
- 用户需要设计系统搭建

## 核心功能

### 1. 设计规范
- 色彩系统
- 字体规范
- 间距系统
- 组件规范

### 2. 设计到代码
- 设计稿解析
- 组件识别
- 代码生成
- 样式转换

### 3. 协作规范
- 命名约定
- 图层组织
- 版本管理
- 交付标准

### 4. 资源复用
- 组件库
- 设计Token
- 模板系统

## 设计规范模板

```markdown
# 设计规范

## 色彩系统
| 名称 | 色值 | 用途 |
|------|------|------|
| Primary | #1890ff | 主色 |
| Success | #52c41a | 成功 |
| Warning | #faad14 | 警告 |
| Error | #f5222d | 错误 |

## 字体规范
| 类型 | 字号 | 字重 | 行高 |
|------|------|------|------|
| H1 | 32px | 600 | 1.2 |
| H2 | 24px | 600 | 1.3 |
| Body | 14px | 400 | 1.5 |

## 间距系统
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px

## 圆角规范
- sm: 4px
- md: 8px
- lg: 12px
- full: 9999px
```

## 工作流程

```
设计稿 → 解析 → 组件识别 → 代码生成 → 优化 → 交付
```

## 输出格式

```markdown
# 设计交付文档

## 一、设计概述
- 页面名称
- 设计目标
- 目标用户

## 二、设计规范
[色彩、字体、间距等]

## 三、组件清单
| 组件 | 类型 | 状态 |
|------|------|------|
| Button | 基础 | 完成 |

## 四、代码输出
[生成的代码]

## 五、注意事项
- ...
```

## 原则

1. **一致性**：统一设计语言
2. **可复用**：组件化思维
3. **可维护**：规范先行
