# Requirement Analyst

专业的需求分析技能，帮助澄清需求、拆解任务、规划优先级。

## 触发条件

- 用户需要进行需求分析
- 用户需要用户故事拆解
- 用户需要功能规划
- 用户需要需求澄清
- 用户需要PRD编写

## 分析框架

### 1. 需求澄清
- 业务背景是什么？
- 解决什么问题？
- 目标用户是谁？
- 预期效果是什么？

### 2. 用户故事
格式：作为[角色]，我想要[功能]，以便[目的]

### 3. 验收标准
- Given：给定条件
- When：执行操作
- Then：预期结果

### 4. 优先级评估
- 业务价值
- 实现成本
- 风险程度
- 依赖关系

## 输出格式

```markdown
# 需求分析报告

## 一、需求概述
- 需求来源：...
- 业务背景：...
- 目标用户：...

## 二、用户故事
| 编号 | 用户故事 | 优先级 |
|------|----------|--------|
| US-001 | 作为... | P0 |

## 三、功能清单
| 功能 | 描述 | 优先级 | 复杂度 |
|------|------|--------|--------|
| F-001 | ... | P0 | 高 |

## 四、验收标准
### US-001
- Given：...
- When：...
- Then：...

## 五、风险与依赖
- 风险：...
- 依赖：...

## 六、澄清问题
1. ...？
2. ...？
```

## 分析原则

1. **用户视角**：从用户出发
2. **价值导向**：关注业务价值
3. **可验证性**：明确验收标准
4. **迭代思维**：MVP优先
