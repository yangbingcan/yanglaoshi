# Brainstorming

创意头脑风暴技能，在实施前充分探索需求和设计。

## 触发条件

- 用户需要创建新功能
- 用户需要构建组件
- 用户需要添加功能
- 用户需要修改行为

## 头脑风暴流程

### 1. 需求探索
- 用户真正想要什么？
- 解决什么问题？
- 目标用户是谁？
- 使用场景有哪些？

### 2. 方案发散
- 列出所有可能方案
- 不评判、不筛选
- 鼓励疯狂想法

### 3. 方案收敛
- 评估可行性
- 比较优缺点
- 选择最佳方案

### 4. 细节设计
- 功能细节
- 交互流程
- 边界情况

## 提问框架

### 5W2H
- What：做什么？
- Why：为什么做？
- Who：给谁用？
- When：什么时候用？
- Where：在哪里用？
- How：怎么做？
- How much：成本多少？

### SCAMPER
- Substitute：替代方案？
- Combine：能否组合？
- Adapt：能否适应？
- Modify：能否修改？
- Put to other use：其他用途？
- Eliminate：能否删除？
- Reverse：能否反转？

## 输出格式

```markdown
# 头脑风暴报告

## 一、需求理解
- 核心需求：...
- 目标用户：...
- 使用场景：...

## 二、方案列表
| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A    | ...  | ...  | ...  |

## 三、推荐方案
- 选择：方案X
- 理由：...

## 四、实施建议
- 第一步：...
- 第二步：...
```

## 原则

1. **先发散后收敛**
2. **不急于下结论**
3. **多角度思考**
4. **用户视角优先**
