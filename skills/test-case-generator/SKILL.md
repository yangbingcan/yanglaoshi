# Test Case Generator

专业的测试用例生成技能，帮助创建全面的测试覆盖方案。

## 触发条件

- 用户需要生成测试用例
- 用户需要设计测试场景
- 用户需要分析边界条件
- 用户需要编写测试计划

## 测试类型

### 1. 功能测试
- 正向用例：正常流程
- 异常用例：错误处理
- 边界用例：临界值

### 2. 界面测试
- 布局验证
- 交互验证
- 兼容性验证

### 3. 性能测试
- 负载测试
- 压力测试
- 并发测试

### 4. 安全测试
- 输入验证
- 权限控制
- 数据保护

## 用例设计方法

### 等价类划分
- 有效等价类
- 无效等价类

### 边界值分析
- 上边界
- 下边界
- 边界外

### 场景法
- 基本流
- 备选流
- 异常流

### 错误推测
- 常见错误
- 历史缺陷

## 输出格式

```markdown
# 测试用例

## 用例编号：TC-001
- **用例名称**：[描述性名称]
- **前置条件**：[执行前需满足的条件]
- **测试步骤**：
  1. 步骤一
  2. 步骤二
  3. 步骤三
- **测试数据**：[输入数据]
- **预期结果**：[期望输出]
- **优先级**：P0/P1/P2/P3
- **测试类型**：功能/界面/性能/安全

---

## 用例汇总
| 用例编号 | 用例名称 | 优先级 | 测试类型 |
|----------|----------|--------|----------|
| TC-001   | ...      | P0     | 功能     |
```

## 测试原则

1. **全覆盖**：功能点、边界值、异常场景
2. **可执行**：步骤清晰、数据明确
3. **可追溯**：关联需求、便于定位
4. **独立性**：用例之间互不依赖
