# Project Planner

专业的项目规划技能，帮助制定完整的项目计划和风险管理方案。

## 触发条件

- 用户需要进行项目规划
- 用户需要制定里程碑
- 用户需要资源分配
- 用户需要项目排期
- 用户需要风险评估

## 规划要素

### 1. 项目范围
- 项目目标
- 交付物清单
- 边界界定
- 排除项

### 2. 里程碑规划
- 关键节点定义
- 交付时间点
- 验收标准
- 依赖关系

### 3. 任务分解 (WBS)
- 工作包划分
- 任务粒度
- 责任分配
- 工时估算

### 4. 资源分配
- 人力资源
- 技术资源
- 预算分配
- 外部依赖

### 5. 风险管理
- 风险识别
- 风险评估
- 应对策略
- 监控机制

## 输出格式

```markdown
# 项目计划书

## 一、项目概述
- 项目名称
- 项目目标
- 项目周期
- 项目团队

## 二、里程碑计划
| 里程碑 | 交付物 | 完成时间 | 负责人 |
|--------|--------|----------|--------|
| M1     | ...    | ...      | ...    |

## 三、任务分解
### 阶段一：[名称]
- [ ] 任务1 - 负责人 - 预计工时
- [ ] 任务2 - 负责人 - 预计工时

## 四、资源计划
- 人力：...
- 技术：...
- 预算：...

## 五、风险管理
| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| ...  | 高/中/低 | 高/中/低 | ... |

## 六、沟通计划
- 日常沟通：...
- 周报机制：...
- 里程碑评审：...
```

## 规划原则

1. **SMART原则**：目标具体、可衡量、可达成、相关、有时限
2. **缓冲预留**：为不确定性预留时间缓冲
3. **持续迭代**：计划随项目进展动态调整
