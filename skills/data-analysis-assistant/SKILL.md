# 数据分析助手

基于用户问题对 Excel 数据进行针对性分析，生成结构完整、结论可靠的 HTML 分析报告。

## 触发条件

当用户有以下需求时，自动调用此 Skill：
- 需要对 Excel 数据进行分析
- 需要生成数据分析报告
- 提出关于数据的问题（如"分析销售趋势"、"会员积分分布如何"等）
- 需要数据可视化或统计摘要

## 工作流程

### 第一阶段：需求澄清

**必须完成此阶段后才能开始分析。**

1. **复述任务**：用 1-3 句话简要复述理解的核心目标与主要问题
2. **主动澄清**：对以下内容有任何不明确，立即提问：
   - 数据源文件路径与工作表名称
   - 分析的具体问题或目标
   - 关键字段含义（如数据字典）
   - 报告的特殊要求（如特定图表类型、重点关注指标）
3. **评估可行性**：若信息不足，明确声明并给出尝试性方案

### 第二阶段：数据探索

1. **读取数据**：使用 pandas 读取 Excel 文件
2. **数据概览**：
   - 数据维度（行数、列数）
   - 字段类型与缺失情况
   - 数据时间范围（如适用）
3. **数据质量检查**：
   - 缺失值统计
   - 异常值识别
   - 重复数据检测

### 第三阶段：针对性分析

根据用户问题，选择适当的分析方法：

| 问题类型 | 分析方法 |
|---------|---------|
| 分布分析 | 频数统计、直方图、饼图 |
| 趋势分析 | 时间序列、折线图、同比环比 |
| 对比分析 | 分组统计、柱状图、箱线图 |
| 相关分析 | 相关系数、散点图、热力图 |
| 构成分析 | 占比计算、堆积图、树图 |

### 第四阶段：生成报告

**标准完整结构**：

```markdown
# [报告标题]

## 一、数据概览
- 数据来源与时间范围
- 数据规模与字段说明
- 数据质量评估

## 二、统计摘要
- 核心指标统计（均值、中位数、极值等）
- 关键维度分布
- 异常数据说明

## 三、可视化分析
- 图表 1：[标题与解读]
- 图表 2：[标题与解读]
- ...

## 四、核心发现
- 发现一：[结论与数据支撑]
- 发现二：[结论与数据支撑]
- ...

## 五、建议与结论
- 业务建议
- 后续分析方向
- 数据局限性说明
```

### 第五阶段：数据验证（强制执行）

**此阶段为强制执行，不可跳过。**

分析完成后，必须执行以下验证流程，确保结果准确可靠：

#### 1. 数据完整性验证

- 验证总记录数是否正确
- 验证各分类数据之和是否等于总数
- 验证分组数量是否与原始数据一致
- 示例：`总记录数 = 已使用 + 未使用 + 作废 + 转赠`

#### 2. 计算准确性验证

- 验证核销率计算：`核销率 = 已使用 / (已使用 + 未使用) × 100%`
- 验证占比计算：`占比 = 分类数 / 总数 × 100%`
- 验证汇总计算：各分组数值之和是否等于整体数值
- 保留小数位数统一，建议保留2位

#### 3. 汇总一致性验证

- 验证分组汇总数据与原始数据逐一核对
- 验证品牌/店铺等维度的汇总是否正确
- 验证交叉汇总（如品牌+使用情况）是否正确
- 示例：`各品牌已使用之和 = 整体已使用`

#### 4. 抽查验证

- 随机抽取3-5条记录与原始数据核对
- 对TOP N数据进行逐一验证
- 对异常数据进行重点核查
- 打印验证结果，标注 ✓ 或 ✗

#### 验证输出格式

```
============================================================
数据验证报告
============================================================
1. 数据完整性验证
   总记录数: 1572 ✓
   分类之和: 255 + 1315 + 1 + 1 = 1572 ✓

2. 计算准确性验证
   整体核销率: 255/(255+1315) = 16.24% ✓
   
3. 汇总一致性验证
   品牌已使用之和: 189+20+31+15 = 255 ✓
   品牌未使用之和: 903+141+223+48 = 1315 ✓

4. 抽查验证
   TOP5店铺数据逐一核对: ✓
   
验证结论: 所有数据准确可靠
============================================================
```

### 第六阶段：输出 HTML 报告

1. 使用 Jinja2 模板引擎生成 HTML
2. 内嵌 ECharts 或 Plotly 交互式图表
3. 应用响应式布局，支持多设备查看
4. 输出文件命名：`数据分析报告_YYYYMMDD_HHMMSS.html`

## 技术规范

### 必用库

```python
import pandas as pd
import numpy as np
import plotly.express as px
from datetime import datetime
```

### 代码规范

1. **数据处理**：
   - 统一使用 pandas 进行数据操作
   - 处理前备份原始数据
   - 记录所有数据转换步骤

2. **可视化**：
   - 优先使用 Plotly 生成交互式图表
   - 图表必须包含标题、轴标签、图例
   - 配色使用专业商务色系

3. **报告生成**：
   - HTML 使用 UTF-8 编码
   - 内联 CSS 样式，确保离线可用
   - 图表数据内嵌，无需外部依赖

## 质量保障

### 分析可靠性

1. **数据验证**：所有统计结果需与原始数据交叉验证
2. **方法合理**：分析方法需与问题类型匹配
3. **结论有据**：每个结论必须引用具体数据支撑
4. **强制验证**：分析结束后必须执行四项验证流程

### 报告完整性

- [ ] 数据概览完整
- [ ] 统计摘要准确
- [ ] 图表清晰可读
- [ ] 结论逻辑连贯
- [ ] 建议切实可行
- [ ] 局限性已说明
- [ ] 数据验证报告已输出
- [ ] 验证结果全部通过（✓）

## 沟通规范

### 诚实表述

- 不确定的分析结果必须标注"推测性结论"
- 禁止使用绝对化承诺
- 数据质量问题时主动告知用户

### 协作处理

当遇到以下情况，请求用户介入：
- 数据字段含义不明确
- 分析结果与业务常识冲突
- 需要领域专业知识判断

## 示例用法

**用户**：分析会员积分分布情况

**助手**：
1. 澄清：确认数据文件路径、积分字段名称、是否需要按会员等级分组
2. 分析：读取数据 → 计算积分分布 → 生成分组统计
3. 输出：包含分布直方图、分组饼图、核心统计指标的 HTML 报告
