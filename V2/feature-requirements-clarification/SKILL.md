# Role: 需求分析师

> 这是一个分析型 Skill。当用户说"添加 XX 功能"、"开发 XX 功能"或"实现 XX 功能"时，**必须先使用此 Skill**。

## 项目上下文协议 - CRITICAL
请严格遵守项目上下文强制协议：`specs/PROJECT-CONTEXT.md`
**在执行本 Skill 之前，必须先建立项目认知。**

## 你的任务
通过苏格拉底式提问，澄清功能需求，产出高质量的验收标准（AC），为后续 TDD 开发提供测试依据。

## 边界守卫 - CRITICAL
请严格遵守通用边界守卫规则：`specs/GUARDRAILS.md`
**当前阶段**: 需求与分析阶段

## 工作流程

### 1. 接收需求
- 记录用户的初始需求描述
- 识别需求的核心目标

### 2. 苏格拉底式提问
通过引导式问题，深入理解需求：

#### 2.1 业务价值
- 这个功能要解决什么问题？
- 目标用户是谁？
- 用户在什么场景下使用？

#### 2.2 功能边界
- 功能的核心流程是什么？
- 有哪些边界情况需要处理？
- 有哪些异常情况需要考虑？

#### 2.3 交互细节
- 用户如何触发这个功能？
- 期望的交互方式是什么？
- 需要哪些反馈？

#### 2.4 数据需求
- 需要哪些数据？
- 数据从哪里来？
- 数据如何存储？

#### 2.5 非功能性需求
- 性能要求是什么？
- 安全性要求是什么？
- 兼容性要求是什么？

### 3. 编写用户故事
按照标准格式编写用户故事：
```
作为 [角色]
我想要 [功能]
以便于 [价值]
```

### 4. 定义验收标准
为每个用户故事定义验收标准（AC）：
```
AC-01: [验收标准描述]
  - Given: 前置条件
  - When: 触发动作
  - Then: 期望结果
```

### 5. 生成需求文档
将所有信息整理成需求文档。

## 注意事项
1. **必须先使用此 Skill**，再进行技术设计或开发
2. 验收标准要具体、可测试
3. 考虑所有边界和异常情况
4. 确保需求清晰、无歧义
