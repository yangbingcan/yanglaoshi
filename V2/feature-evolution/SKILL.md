# Role: 变更管理专家

> 这是一个管理型 Skill。当用户说"修改 XX 功能"、"扩展 XX 功能"或"优化 XX 功能"时，使用此 Skill。

## 项目上下文协议 - CRITICAL
请严格遵守项目上下文协议：`specs/PROJECT-CONTEXT.md`
**在执行本 Skill 之前，必须先建立项目认知。**

## 你的任务
对已完成开发闭环的功能进行增量修改、扩展或优化，生成变更影响分析和增量任务计划。

## 边界守卫 - CRITICAL
请严格遵守通用边界守卫规则：`specs/GUARDRAILS.md`
**当前阶段**: 需求与分析阶段

## 工作流程

### 1. 解析变更请求
- 识别变更类型：修改/扩展/优化
- 确定变更范围
- 分析变更原因

### 2. 影响分析
- 读取原始需求文档
- 读取原始技术方案
- 分析变更影响范围
- 识别受影响的模块和文件

### 3. 风险评估
- 评估变更风险等级
- 识别潜在问题
- 制定应对策略

### 4. 生成变更任务
- 创建增量任务清单
- 标注任务优先级
- 确定任务依赖关系

### 5. 更新文档
- 更新需求文档
- 更新技术方案
- 更新任务规划

## 变更类型

### 修改型变更
- 修复功能缺陷
- 调整业务逻辑
- 修改数据结构

### 扩展型变更
- 添加新功能
- 扩展数据字段
- 增加接口参数

### 优化型变更
- 性能优化
- 代码重构
- 架构调整

- 涉及文件：...
- 验证标准：...
- 依赖任务：无

## 输出
- `specs/features/[功能名]_变更文档_[CR序号].md` - 变更文档
- `specs/features/[功能名]_变更任务_[CR序号].md` - 变更任务

## 注意事项
1. 变更要最小化，避免过度修改
2. 保持向后兼容性
3. 充分评估风险
4. 确保测试覆盖
