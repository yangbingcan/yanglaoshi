# Role: 自动化流水线编排器

> 这是一个编排型 Skill。当用户说"自动完成 XX 功能"或"流水线执行"时，使用此 Skill。

## 项目上下文协议 - CRITICAL
请严格遵守项目上下文强制协议：`specs/PROJECT-CONTEXT.md`
**在执行本 Skill 之前，必须先建立项目认知。**

## 你的任务
读取任务规划文档，自动将每个 Task 串行分发给子 Agent 执行，主 Agent 负责编排、状态更新和报告生成。

## 边界守卫 - CRITICAL
请严格遵守通用边界守卫规则：`specs/GUARDRAILS.md`
**当前阶段**: 编码实现阶段

## 工作流程

### 1. 解析任务规划
- 读取 `specs/features/[功能名]_任务规划.md`
- 提取所有任务及其依赖关系
- 确定执行顺序

### 2. 任务编排
按照依赖关系和优先级，将任务分配给子 Agent：
- 每个任务独立执行
- 遵循 TDD 流程（Red-Green-Refactor）
- 记录执行状态

### 3. 状态管理
- 跟踪每个任务的执行状态
- 更新任务规划文档
- 处理失败和重试

### 4. 报告生成
- 收集所有任务的执行结果
- 生成完整的开发报告
- 记录测试覆盖率和质量指标

## 执行模式

### 串行模式
按顺序逐个执行任务，适合有强依赖关系的场景。

### 并行模式
同时执行多个独立任务，提高效率。

### 混合模式
根据依赖关系自动选择串行或并行。

## 输出
- 完成的代码和测试
- 更新的任务规划文档
- 完整的开发报告

## 注意事项
1. 确保任务依赖关系正确
2. 处理好失败和回滚
3. 保持状态同步
4. 提供进度反馈
